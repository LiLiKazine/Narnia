# 2026-06-22-07: Quick-exit / panic (explicit in-app exit)

**Status:** Implemented — explicit-exit portion of roadmap item "Quick-exit / panic". Flip-face-down-to-lock deferred.

## Context

Spec §5 listed quick-exit as "not yet designed", and it became more relevant once
§3's fresh-start model meant backgrounding no longer ejects to the cover. This
slice designs + builds an explicit, thumb-reachable panic exit. Iteration 8 of the
autonomous dev loop (team lead + 1 impl agent + review).

## Decision (team-lead, spec was silent)

An explicit exit control inside the vault that re-locks the session and returns to
the cover. Key realization: the vault lives behind `if session.isUnlocked { NavigationStack{…} } else { CoverView }`,
so flipping the session to locked swaps the ENTIRE vault subtree out — which by
construction collapses all folder depth AND dismisses any presented viewer. So
quick-exit = `VaultSession.lock()` + a reachable control; no need to lift the
navigation path.

`lock()` is the DELIBERATE user-initiated exception to the otherwise one-way
session. It does NOT contradict §3's rejection of *automatic* relock (on
background/return) — that automatic path remains one-way (`unlock()` unchanged,
no scenePhase handlers). Only the explicit button calls `lock()`.

## What Changed

- `VaultSession.swift` — added `func lock() { isUnlocked = false }` (explicit exit only).
- `ContentView.swift` — a thumb-reachable `.overlay(alignment: .bottomTrailing)` exit button (xmark.circle.fill, `vaultExitButton`) on the vault `NavigationStack`, calling `session.lock()`.
- `HardenedVideoView.swift` — added `dismantleUIViewController` that pauses + detaches the `AVPlayer` on teardown, so exit/closing the viewer deterministically STOPS playback (and audio) rather than trusting ARC timing (review-flagged; satisfies §5 "stops any playing video").
- `VaultSessionTests.swift` — `lock()` tests: unlock→lock→false, idempotent, lock-then-unlock re-auths.
- `VaultGridUITests.swift` — `testQuickExitReturnsToCover` (autounlock → tap exit → cover reappears, grid gone via `waitForNonExistence`).
- `README.md` — ticked Quick-exit (flip-to-lock noted deferred).

## What Was Discovered

- **The root swap does the heavy lifting** — collapsing folder depth and dismissing the viewer are automatic consequences of removing the vault subtree; quick-exit needed almost no navigation plumbing.
- **"Stops playing video" needed explicit teardown** — review caught that the root swap relied on ARC to release the `AVPlayer`, so audio could briefly continue. Added a deterministic pause-on-dismantle (also fixes a latent video-viewer teardown gap from the viewer slice).
- **Known gap (deferred):** the exit overlay sits on the NavigationStack, so while a full-screen viewer (`.fullScreenCover`) is open the panic button is covered — you'd close the viewer (its own close button) first. In the current reachable flow exit fires from the grid (no video playing). A follow-up could add an exit affordance inside the viewer or drive viewer dismissal off `session.isUnlocked`.
- **Deferred:** opt-in flip-face-down-to-lock (CoreMotion) — its own sub-slice; and a Realm settings home for the exit/flip prefs.
