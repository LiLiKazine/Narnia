# 2026-06-22-09: "Hide names" toggle (caption privacy)

**Status:** Implemented — roadmap item "Hide names toggle". Completes the core roadmap.

## Context

Spec §5: a toggle to mitigate the on-screen-text leak introduced by captions (§1
shows folder/document names; for documents the caption is the sole identifier).
Iteration 10 of the autonomous dev loop (team lead + 1 impl agent + review). Its
home is the Realm settings screen built in -08.

## Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Storage | `@AppStorage("vault.hideNames")` (Bool, default false) | Read by the settings toggle, the cell, and the grid title without threading an object; shared + observed via UserDefaults |
| Caption logic | pure `VaultItemCell.displayCaption(name:kind:hideNames:)` | Unit-testable without UI; folder/document → name (off) or "Folder"/"Document" (on); photo/video/other → nil always |
| Nav title | genericized to "Folder" when hidden (root stays "Vault") | The folder name would otherwise leak at the top |
| A11y label | genericized when hidden (review fix) | The spoken VoiceOver/automation label must not leak the real name either |
| Documents tradeoff | accepted, documented | A document's caption is its sole identifier (§1), so hide-names deliberately makes documents unidentifiable — the opt-in cost |

## What Changed

- `VaultItemCell.swift` — `@AppStorage` + pure `displayCaption` helper; caption renders the helper's result (or the existing hidden placeholder for nil, keeping heights uniform).
- `SettingsView.swift` — un-stubbed the "Security" section with a "Hide names" `Toggle` (`hideNamesToggle`) + tradeoff footer.
- `VaultGridView.swift` — `@AppStorage`; `navigationTitle` genericizes folder names when hidden; **and the cell `accessibilityLabel` genericizes when hidden** (was leaking `item.name` to VoiceOver/automation — caught in review).
- `VaultItemCellTests.swift` (new) — 7 cases over `displayCaption`.
- `VaultGridUITests.swift` — a light test that the toggle is present/operable.
- `README.md` — ticked Hide names.

## What Was Discovered

- **The leak isn't only visual.** Review caught that genericizing the visible caption + nav title still left `accessibilityLabel(item.name)` exposing the real name to VoiceOver and UI automation — undermining the very §5 mitigation. Fixed by deriving the spoken label from the same hide-names state. The `item-<name>` accessibilityIdentifier stays (not spoken; the UI test needs it).
- **@AppStorage is the right fit** for a simple shared bool read in many places (cell, settings, title) — no object threading, automatic persistence + observation.
- A pure static caption helper kept the privacy logic fully unit-tested even though it drives a view.
